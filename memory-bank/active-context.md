# Актуальный контекст задачи

**Задача:** Обеспечить корректное создание и восстановление кошелька, а также проверить тесты после исправления проблемы с расхождением мнемоник.

**Проблема:** При создании кошелька пользователю показывалась одна seed-фраза, а сохранялась другая. Функция `startCreating()` генерировала мнемонику для UI, а `createNewWallet()` генерировала новую повторно и шифровала уже её. Восстановление созданного кошелька по показанной фразе было невозможно.

**Принятое решение:** Исправлено в `src/components/views/Auth.vue`, чтобы `createOrImportWallet` использовал мнемонику, отображаемую пользователю (`mnemonic.value`), при создании кошелька.

**Состояние тестов:**
- Все тестовые файлы перемещены в `tests/unit/`.
- `tests/unit/crypto.service.spec.ts`: Окружение `node`, тесты криптографических операций, использует реальные сервисы.
- `tests/unit/ton.service.spec.ts`: Окружение `node`, тесты TON-сервисов, использует реальные сервисы.
- `tests/unit/wallet-manager.store.spec.ts`: Окружение `node`, тесты логики хранилища, использует реальные сервисы `cryptoService` и `@ton/crypto`.
- `tests/unit/Auth.ui.spec.ts`: Окружение `jsdom`, UI-тесты компонента `Auth.vue` с полным моком `useWalletManagerStore`.
- `tests/unit/verify-creation-mnemonic.spec.ts` был удален.

**Следующие шаги:** Запустить все тесты, чтобы убедиться в их успешном прохождении после всех внесенных изменений.
